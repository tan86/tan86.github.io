SOLID Principles

Single Responsibility Principle

  • In short, SRP means each code unit should have exactly one responsibility.
  • This means writing functions that do one thing only, creating types that are responsible for a single thing.
  • There should never be more than one reason for a class to change.
  • A related topic is the principle of least knowledge. In essence, it says that no object should know no more than necessary about other objects, so it doesn't depend on any of their internals.

    • Open-Closed Principle

  • The OCP means that code should be open for extensions but closed for modifications.
  • Open for extension means that we could extend the list of types the code supports easily. Closed for modification means existing code shoudn't change, as this can often cause bugs somewhere else in the system.
  • A great feature of C++ demonstrating this principle is `operator<<` of `ostream`.
